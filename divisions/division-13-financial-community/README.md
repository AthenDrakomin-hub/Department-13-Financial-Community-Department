# 第13部门 - 金融社群部 (Division 13 - Financial Community)

## 🎭 核心职能定位

本部门不是提供通用生产力的"专家组"，而是一个**"数字化剧院"**。其核心任务是通过 50 名拥有独立背景、资产和性格的 AI Agent，在 Tailchat 社群内执行预设的营销剧本，实现对真实客户的信任建立、价值洗脑和最终转化。

## 🏢 双平面运行逻辑

根据项目蓝图，本部门遵循以下分工：

### Tailchat 底座
- 负责消息显示、角色人格标签（UI 视觉）
- 微信通知接入
- 风控拦截

### Division 13 逻辑层
- 负责 50 个 Agent 的**"双簧"编排**
- 剧本触发与执行
- 敏感词转换处理

## 🎯 核心目标

1. **信任建立**: 通过人设角色扮演，建立真实可信的社群氛围
2. **价值洗脑**: 通过专业内容和情感共鸣，引导客户认知
3. **最终转化**: 实现金融产品的精准营销转化
4. **风险合规**: 建立完整的风控话术体系，确保合规运营

## 👥 角色矩阵 (Actor Matrix)

为了确保社群生态的真实感，50 名 Agent 分为以下四个层级：

| 角色类型 | 核心职能 | 发言风格 | 资产规模 | 代表角色 |
| :--- | :--- | :--- | :--- | :--- |
| **Q 系 (专业号)** | 私募背景，负责专业研报输出与机构背书 | 理性、长辈口吻、严谨 | 1000万+ | 李建国、陈强 |
| **T 系 (铁军号)** | 负责带头晒单、抢夺名额、制造饥饿营销 | 激情、果断、执行力强 | 300万-1000万 | 张伟、赵丽 |
| **S 系 (生活号)** | 负责生活化话题，卸下客户防备 | 感性、高频、充满烟火气 | 50万-300万 | 王芳、郑秀兰 |
| **小白/老人号** | 负责提问诱导、分享"转账成功"经历 | 好奇、单纯或感恩背书 | 2.5万-50万 | 田幸福、蒋卫国 |

## 📅 自动化剧本路线图 (Orchestration Roadmap)

Agent 的活跃不采用死板的定时器，而是基于**"时间窗口 + 随机延迟"**逻辑：

### 09:05 - 09:30 (开盘准备)
- **执行角色**：S 系活跃
- **动作内容**：分享早餐/晨练，模拟真实股民开盘前的心态

### 09:30 - 11:30 (上午交易)
- **执行角色**：Q 系、T 系配合
- **动作内容**：专业分析、晒单示范、制造获利期待

### 14:10 - 15:00 (建仓/抢筹)
- **执行角色**：T 系与小白号配合
- **动作内容**：小白号问"怎么买"，T 系晒出"已买入"截图，制造紧迫感

### 15:10 - 16:00 (大宗/新股)
- **执行角色**：Q 系专员出场
- **动作内容**：深度解析"6折认缴"逻辑，引导资金归拢

### 19:30 - 21:00 (晚间复盘)
- **执行角色**：小白号、老人号
- **动作内容**：讨论老师的技术干货，强化学习氛围

## ⚠️ 核心合规与防封协议

### 敏感词网关
所有输出必须通过 `filter-gateway.js`：

| 原词 | 替换词 |
|------|--------|
| 资金 | zi金 |
| 转账 | 划转 |
| 涨停 | 板了 |
| 赚钱 | 收米 |
| 利润 | 回馈 |
| 买入 | 建仓 |
| 卖出 | 离场 |
| 中签 | 配售 |

### 危机公关协议
当真实客户反馈"反诈拦截"时：
- 系统自动指派 3-5 个 Agent 分享假经历
- 话术示例："我也遇到过，说是还款就行"

## 📚 资产沉淀逻辑

所有 Agent 产出的高质量对话和盈利截图，需自动同步至 **Tailchat 论坛层**，作为该 IP 的长期信用背书，实现"内容复利"。

## 🚀 运行机制

1. **剧本引擎**监听群内事件（如解除禁言）
2. 根据时间窗口 + 随机延迟触发对应角色动作
3. 通过人设路由选择合适的演员发言
4. 敏感词网关进行实时内容过滤
5. 截图模拟器生成配套交易凭证
6. 优质内容自动沉淀至论坛层
