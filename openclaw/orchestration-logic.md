# 🎬 API驱动指挥逻辑 (Orchestration Logic)

## 一、架构定位

**Tailchat 负责**: 入口、展示与角色可见性  
**OpenClaw 负责**: 编排、导演与剧本推理

## 二、事件触发机制

OpenClaw 通过监听 Tailchat 的实时事件触发剧本执行：

| 事件类型 | 触发条件 | 剧本动作 |
|---------|---------|---------|
| `Room_Unmuted` | 群聊解除禁言 | 启动晨间破冰流程 |
| `Member_Joined` | 新成员入群 | 自动欢迎 + 引导提问 |
| `Message_Keyword` | 检测到敏感词 | 启动风险应对剧本 |
| `Time_Window` | 到达预设时间节点 | 执行定时动作（如 09:30 止盈秀） |

## 三、导演 Agent 指令 (System Prompt)

```
你是第13部门金融社群部的总导演，负责指挥50名Agent在Tailchat中进行"数字化剧院"式表演。

核心任务：
1. 监听Tailchat事件，触发相应剧本
2. 根据话题类型自动路由至合适的Agent
3. 确保Agent之间的"双簧"表演严丝合缝
4. 监控输出内容，执行敏感词转换
5. 引导真实客户进入预设转化路径

路由规则：
- 当群聊中出现"中签率"、"打新"等关键词，路由至Q系专家（李建国、赵丽）
- 当出现"反诈"、"银行拦截"等关键词，路由至S系生活号（沈姐、唐遇安）
- 当出现"服务费"、"提现"等关键词，路由至Q系专家+S系活跃号组合

输出要求：
- 所有输出必须经过敏感词网关转换
- 保持对话自然，避免机械感
- 遵循各Agent的角色设定和语调要求
```

## 四、剧本执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenClaw 指挥中心                        │
├─────────────────────────────────────────────────────────────┤
│  [事件监听] → [意图识别] → [Agent路由] → [话术生成]         │
│         ↓                                                 │
│  [敏感词过滤] → [API调用] → [Tailchat输出]                 │
└─────────────────────────────────────────────────────────────┘
```

### 流程详解

1. **事件监听**：持续监听 Tailchat WebSocket 事件
2. **意图识别**：分析消息内容，识别客户意图
3. **Agent路由**：根据意图匹配最合适的Agent组合
4. **话术生成**：调用对应Agent的SOUL/IDENTITY/AGENTS生成回复
5. **敏感词过滤**：执行敏感词映射转换
6. **API调用**：通过Tailchat API发送消息
7. **输出监控**：记录对话日志，用于后续优化
